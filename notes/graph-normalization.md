|Method              |Pros                                                                                                                    |Cons                                                                                                                               |Difficulty                   |
|--------------------|------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------|-----------------------------|
|Bag of Nodes (BoN)  |Dead simple. Extremely fast to code. Interpretable results (you can see exactly which node counts triggered a cluster). |Topology-blind. It doesn't know how nodes are connected. Two very different programs could theoretically have the same node counts.|Low (Pandas + JSON)          |
|Graph2Vec           |Captures Context. Sees "neighborhoods" around nodes. Robust against variable renaming and minor structural shuffling.   |Lossy. Compresses the graph into a fixed-length vector. Requires a learning phase (embedding) before clustering.                   |Medium (Requires karateclub) |
|WL Features (Kernel)|Mathematically Rigorous. Practically lossless for structural isomorphism. Perfect for academic defense (highly precise).|Computational Cost. Can be slow on very large graphs. Distance matrices are slightly more annoying to manage than feature tables.  |Medium-High (Requires grakel)|
